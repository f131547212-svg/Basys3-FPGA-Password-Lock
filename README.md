# 基於邊緣觸發與狀態機控制之 FPGA 互動式硬體密碼鎖系統

## 1. 專題名稱
基於純 Verilog 邊緣觸發與有限狀態機（FSM）控制之 FPGA 互動式動態七段顯示器硬體密碼鎖系統。

## 2. 使用開發板
* **開發板型號：** Digilent Basys 3 (搭載 Xilinx Artix-7 FPGA, 晶片型號：`xc7a35tcpg236-1`)。

## 3. 使用工具版本
* **開發軟體：** Xilinx Vivado 2022.2 (用於綜合、實現與 Bitstream 生成)。
* **硬體描述語言：** Verilog-2001 標準。
* *(註：本專題最終優化版為純硬體狀態機控制，未涉及 RISC-V 軟核組合語言指令編譯，故無需配置外部 RISC-V Toolchain 版本。)*

## 4. 專案資料夾結構
本專案原始碼與管腳約束檔案結構如下：
```text
├── top.v           # 系統頂層核心：包含按鈕正向邊緣偵測與密碼鎖有限狀態機 (FSM)
├── seg7_driver.v    # 七段顯示器驅動：內含 20-bit 動態掃描分頻計數器與字型解碼器
├── mini_rv32i.v    # 單週期 RISC-V CPU 核心基礎架構（保留作軟硬體整合觀念演練參考）

├── addr_decoder.v  # 位址解碼器模組
└── basys3_pins.xdc # 實體管腳約束檔：指定 Basys 3 開發板之開關、按鈕、LED 與七段顯示器實體腳位

## 5. 如何產生 Bitstream
1. 開啟 **Xilinx Vivado 2022.2** 軟體，並建立一個新專案（RTL Project），晶片選取 `xc7a35tcpg236-1`。
2. 將 `top.v`、`seg7_driver.v`、`mini_rv32i.v`、`imem.v` 與 `addr_decoder.v` 全部加進專案的 **Design Sources**。
3. 將 `basys3_pins.xdc` 加進專案的 **Constraints**。
4. 在左側 Flow Navigator 導覽列中，點擊最下方的 **【Generate Bitstream】**。
5. 點擊 **【OK】**，等待系統自動執行完綜合（Synthesis）與實現（Implementation），直到右上角跳出完成提示。

## 6. 如何載入或修改 RISC-V 程式
*(硬體邏輯優化說明)*
本專題之原始架構規劃透過 RISC-V 處理器讀取 `imem.v` 並以輪詢方式讀取按鈕。但在實機 100MHz 高頻運作下，非同步按鈕在無去抖動處理下會造成 CPU 在極短時間內產生誤判鎖死。

## 7. 如何燒錄到 FPGA 開發板
1. 使用 Micro-USB 線將 **Basys 3 開發板** 連接至電腦，並將板上的電源開關切至 **【ON】**。
2. 當 Vivado 產生 Bitstream 成功後，在彈出的對話框中選擇 **【Open Hardware Manager】**（或點選左下角選單最下方的 Open Hardware Manager）。
3. 點擊畫面上方綠色橫條中的 **【Open target】** $\rightarrow$ 選擇 **【Auto Connect】**。
4. 連線成功後，點擊綠色橫條中的藍色字體 **【Program device】**。
5. 在跳出的對話框中確認選取了剛剛產生的 `.bit` 檔案，點擊右下角的 **【Program】** 進行燒錄（當板載右上角綠色 **DONE** 燈亮起，代表燒錄成功）。

## 8. 如何操作與測試
系統燒錄完畢後會自動啟動，請依照以下步驟進行互動式測試：
* **初始狀態：** 四位數七段顯示器會穩定顯示橫線 **`----`**。
* **正確解鎖：** 將最右側 4 顆指撥開關調整為 `下、上、下、上`（對應二進位 `0101`），接著按下板子正中央的按鈕 **BTN-C**，七段顯示器將立即完美亮起 **`PASS`**。
* **錯誤判定：** 將開關調整為其餘任意錯誤密碼，按下 **BTN-C**，顯示器會亮起 **`FAIL`**。
* **鎖死機制：** 故意連續按錯達 3 次時，系統判定遭到入侵，七段顯示器將強制顯示 **`LOCK`**，且開發板最左側的 LED 燈（**LD15**）會同步亮起。此時系統進入安全鎖死狀態，撥動任何開關或按壓 BTN-C 皆無反應。
* **系統重置：** 隨時按下開發板 5 顆按鈕中最右側的 **BTNR (Reset)** 鈕，系統即會非同步清除錯誤計數與鎖死狀態，回復至初始 `----` 狀態。

## 9. 已知問題
* **動態掃描微弱鬼影：** 由於板端時脈極高（100MHz），在進行七段顯示器快速動態掃描切換時，未亮起之 LED 管腳可能因殘留微弱餘暉產生鬼影。本專案已透過優化 `seg7_driver.v` 中的 20 位元動態掃描分頻計數器（取高兩位 `refresh_counter[19:18]` 進行位數切換）將其降至最低，完全不影響整體功能判定。
